hexo+git 搭建博客

hexo

这里主要是把我自己建立博客的过程记录一下，主要用到的git和hexo。git是一个开源的分布式版本控制系统（主要是免费，而且上面的项目多，多用点没坏处）
hexo是一个开源的静态博客生成器，用node.js开发。系统window7

  • git下载
  • node.js下载

我下载的git版本是 2.8.2.window.1、node.js 版本是v4.4.4LTS了,因为版本问题在配置hexo要同步到git的时候有点不太一样，这个也是我遇到的一个问题
git和node.js的安装就简单的下一步，这里就不说了
安装完成后在本地生产git的ssh验证，用来完成验证

检查SSH Keys的设置

首先我们需要检查电视上现有的ssh key:

$ cd ~/.ssh

如果提示：no such file or directory说明你是第一次使用git

生成新的SSH Key:

$ ssh-keygen -t rsa -C “邮箱地址”

注意：这里的邮箱地址，填写git注册的邮箱地址，[-C] 这里的C是大写
输入上面的命令后会出现如下两行：

Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

回车之后新系统会让你输入密码：

Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

注意：输入密码的时候没有*字样的，你直接输入就可以了。
在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
可能不填需要多次回车，网上看的好像并没有出现我这情况，我就是一直回车了。
最后我们就会有看到如下的界面

添加SSH Key到GitHub

在本机设置SSH Key之后，需要添加到gitHub上，已完成SSH链接的设置

  • 打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
  • 登录github系统，进入用户设置找到一个有ssh key的设置将本地生成的秘钥复制进文本框（这里因为git升级，所以界面可能不一样，不过ssh设置还是很好找了）

检查SSH Key是否成功

输入下面命令，git@github.com的部分不要修改：

ssh -T git@github.com

如果是下面的反馈：

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

输入yes，然后就会看到：

Hi cnfeat! You’ve successfully authenticated, but GitHub does not provide shell access.

这样就说明成功了。

设置用户信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。

Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，而不是GitHub的昵称。

github上建立仓库

登录github后,找到「New Repository」,新建的仓库名称需要跟你的账号名一样，例如：github账号名是example,那么新建的仓库名就是example.github.io(这个是之后访问博客的域名，除非自己买新的域名绑定)

安装hexo

打开git,输入下面命令。

$ npm install -g hexo

hexo初始化

在电脑新建一个文件夹然后右键Git Bash,输入命令

$ hexo init

之后文件夹会自动生成一些文件，之后输入如下命令可以查看

$ hexo g
$ hexo s

这样就可以通过localhost:4000查看了。

hexo关联git

首先安装hexo git插件

npm install hexo-deployer-git –save

然后自己新建的文件夹下找到_config.yml，打开找到原先的

deploy:
　type:

修改为

deploy:
　type: git
　repo: 这里写入git新建仓库后显示的ssh地址

网上看的有的type是修改为github,repo是repository。之前我这么设置就一直没法发布到git上，这个因为与版本有关，具体就不太清楚了。
最后输入

$ hexo d -g

第一次可能会弹出输入git账号密码，之后就不需要了，还有就是这步骤的命令因为要下载和上传git要久一点，耐心等待。

最后就可以通过你新建仓库的名称来访问你的博客了。
