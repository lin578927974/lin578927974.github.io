SSH框架搭建之Spring

之前的博客已经搭建好的Struts2和Hibernate了，现在在此基础上搭建Spring。因为官网的修改所有Spring不好找，不过Spring有专门的下载地址如下：

  • Spring

我现在的版本是spring-framework-4.2.6.RELEASE，将下载的解压拷贝lib下的jar到项目中（不用拷贝有javadoc和sources了），同时拷贝Hibernate下lib目录下optional\c3p0的mchange-commons-java-0.2.3.4.jar和Struts目录lib下的commons-logging-1.1.3.jar和struts2-spring-plugin-2.3.28.1.jar。这三个是我根据网上的教程搭建完后发现有报错，查看错误日志后导入的包，不用像网上哪有导入一堆的包，多看下输出的错误信息，根据错误信息去找包或者修改，不要都跟着网上来。
导入包以后，项目结构也要有所修改：
1.编写Spring的配置文件applicationContext.xml。路径：src目录下，需要在web.xml配置context-param指定路径，或者把该文件放在WEB-INF下，跟web.xml同目录。这里由于Spring配置数据源的需要，需要把Hibernate内lib/optional/c3p0下的c3p0-0.9.1.jar复制到lib不目下。

2.修改BaseDao和UserDao。在引入Spring后，需要用Spring进行统一的事务管理，数据源和sessionFactory都交给Spring去生成，因此接口类和实现类BaseDao和UserDao都需要做相应的修改。Spring提供了HibernateDaoSupport类来完成对数据的操作，因此UserDao在实现BaseDao的同时还需要继承HibernateDaoSupport类。并将先前session的操作修改成HibernateTemplate（可通过getHibernateTemplate（）方法来获得）的操作。

  • BaseDao修改
  • UserDao修改

3.修改业务逻辑实现类。在没有加入Spring之前，业务逻辑实现类的Session的获得，dao的实例化，以及事务的管理都是该类执行管理的。加入Spring后，这些都交给Spring去管理。该类的dao的实例化由Spring注入。

4.修改用户注册的RegisterAction类。同样，RegisterAction类中的userManager的实例化也由Spring注入。

5.删除Hibernate的配置文件Hibernate.cfg.xml和工厂类HibernateSesseionFactory类。他们的工作已经交给Spring去做，已经不再有用。
6.修改web.xml，加载Spring。要想启动时加载Spring的配置文件，需要在web.xml中配置对应的监听器（listenser），并制定Spring的配置文件。

7.修改Struts的配置文件struts.xml。把原来指定的名为register的action的class由原来的路径变为applicationContext.xml文件中该Action的id。

以上就搭建好的SSH框架了，之后还要慢慢深入了解下，现在只是知道一些基础了。

使用之前

因为Burp Suite需要使用Java环境，所以需要jdk的安装配置。之后会写一个单独的博文，待更新。

Burp Suite介绍

Burp Suite是用于攻击web引用程序的集成平台。它包含了许多工具，并为这些工具设计了许多借口，以促进加快攻击应用程序的过程。所有的工具都共享一个能处理并显示HTTP消息，持久性，认证，代理，日志，警报的一个强大的可拓展的框架。本文主要介绍它的一下特点：

  • Traget(目标)–显示目标目录结构的一个功能
  • Proxy(代理)–拦截HTTP/S的代理服务器，作为一个在浏览器和目标应用程序直接的中间人，允许你拦截，查看，修改在两个方向上的原数据流。
  • Spider(蜘蛛)–应用智能感应的网络爬虫，他能完整的枚举应用程序的内容和功能。
  • Scanner(扫描器)–高级工具，执行和，他能自动地发现web应用程序的安全漏洞。
  • Intruder(入侵)–一个定制的高度可配置的工具，对web应用程序进行自动化攻击，如：枚举标识符，收集有用的数据，以及使用fuzzing技术探测常规漏洞。
  • Repeater(中继器)–一个靠手动操作来触发单独的HTTP请求，并分享应用程序响应的工具。
  • Sequencer(会话)–用来分享那些不可预知的应用程序会话令牌和重要数据项的随机性的工具。
  • Decoder(解码器)–用来手动执行或对应用程序数据者智能解码编码的工具。
  • Comparer(对比)–通常是通过一些相关的请求和响应得到两项数据的一个可视化的“差异”。
  • Extender(扩展)–可以让你加载Burp Suite的扩展，使用你自己的或第三方代码来扩展Burp Suite的功能。
  • Options(设置)–对Burp Suite的一些设置

测试工作流程

Burp支持手动的Web应用程序测试的活动，他可以让你有效地结合手动和自动化技术，使你可以完全控制所有的BurpSuite执行的行动，并提供有关您所测试的应用程序的详细信息和分析。如下图：

代理工具可以说是Burp Suite测试流程的一个心脏，它可以让你通过浏览器来浏览应用程序来捕获所有相关信息，并让您轻松地开始进一步行动，在一个典型的测试中，侦察和分析阶段包括以下任务：

手动映射应用程序-使用浏览器通过BurpSuite代理工作，手动映射应用程序通过以下链接，提交表单，并通过多步骤的过程加强。这个过程将填充代理的历史和目标站点地图与所有请求的内容，通过被动蜘蛛将添加到站点地图，可以从应用程序的响应来推断任何进一步的内容(通过链接、表单等)。也可以请求任何未经请求的站点(在站点地图中以灰色显示的)，并使用浏览器请求这些。

在必要是执行自动映射-您可以使用BurpSuite自动映射过程中的各种方法。可以进行自动蜘蛛爬行，要求在站点地图未经请求的站点。请务必在使用这个工具之前，检查所有的蜘蛛爬行设置。

注意，在执行任何自动操作之前，可能有必要更新的BurpSuite的配置的各个方面，诸如目标的范围和会话处理。

分析应用程序的攻击面 - 映射应用程序的过程中填入代理服务器的历史和目标站点地图与所有的BurpSuite已抓获有关应用程序的信息。这两个库中包含的功能来帮助您分析它们所包含的信息，并评估受攻击面的应用程序公开。此外，您可以使用BurpSuite的目标分析器报告的攻击面的程度和不同类型的应用程序使用的URL 。

MarkDown介绍

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。
使用MarkDown优点

  • 专注你的文字内容而不是排版样式。
  • 轻松的导出 HTML、PDF 和本身的 .md 文件。
  • 纯文本内容，兼容所有的文本编辑器与字处理软件。
  • 可读，直观。适合所有人的写作语言。

MarkDown使用

常用样式

  • 换行符：一行结束时输入两个空格
  • 文本缩进：用全角输入空格
  • 转义字符：\,通过转义输入要不的标签

使用一些MarkDown后最好是回车一下，不然文本段落会连续在一起无法区分。

标题

这是最为常见的格式，在平常是使用的文本编辑器大多是这样实现的：输入文本、选中文本、设置标题格式。
在MarkDown中，你只需要在文本前面加上 # 即可，同理你可以增加二级标题、三级标题、四级标题、五级标题和六级标题，总共六级。例如：

二级标题

五级标题
六级标题

列表

列表格式也很常用，在MarkDown中有两种列表格式，一种有序和一种无序。
无序列表，只要在文字前面加上 - 就可以了，例如：

  • 文本1
  • 文本2
  • 文本3

有序列表，只要在文本前面1. 2. 3. 就可以了，例如：

  1. 文本1
  2. 文本2
  3. 文本3

注：-、1. 和文本之间要保留一个字符的空格。

hexo+git 搭建博客

安装完成后在本地生产git的ssh验证，用来完成验证

检查SSH Keys的设置

首先我们需要检查电视上现有的ssh key:

$ cd ~/.ssh

如果提示：no such file or directory说明你是第一次使用git

生成新的SSH Key:

$ ssh-keygen -t rsa -C “邮箱地址”

注意：这里的邮箱地址，填写git注册的邮箱地址，[-C] 这里的C是大写
输入上面的命令后会出现如下两行：

Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

回车之后新系统会让你输入密码：

Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

注意：输入密码的时候没有*字样的，你直接输入就可以了。
在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
可能不填需要多次回车，网上看的好像并没有出现我这情况，我就是一直回车了。
最后我们就会有看到如下的界面

添加SSH Key到GitHub

在本机设置SSH Key之后，需要添加到gitHub上，已完成SSH链接的设置

  • 打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
  • 登录github系统，进入用户设置找到一个有ssh key的设置将本地生成的秘钥复制进文本框（这里因为git升级，所以界面可能不一样，不过ssh设置还是很好找了）

检查SSH Key是否成功

输入下面命令，git@github.com的部分不要修改：

ssh -T git@github.com

如果是下面的反馈：

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

输入yes，然后就会看到：

Hi cnfeat! You’ve successfully authenticated, but GitHub does not provide shell access.

这样就说明成功了。

设置用户信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。

Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，而不是GitHub的昵称。

github上建立仓库

登录github后,找到「New Repository」,新建的仓库名称需要跟你的账号名一样，例如：github账号名是example,那么新建的仓库名就是example.github.io(这个是之后访问博客的域名，除非自己买新的域名绑定)

安装hexo

打开git,输入下面命令。

$ npm install -g hexo

hexo初始化

在电脑新建一个文件夹然后右键Git Bash,输入命令

$ hexo init

之后文件夹会自动生成一些文件，之后输入如下命令可以查看

$ hexo g
$ hexo s

这样就可以通过localhost:4000查看了。

hexo关联git

首先安装hexo git插件

npm install hexo-deployer-git –save

然后自己新建的文件夹下找到_config.yml，打开找到原先的

deploy:
　type:

修改为

deploy:
　type: git
　repo: 这里写入git新建仓库后显示的ssh地址

网上看的有的type是修改为github,repo是repository。之前我这么设置就一直没法发布到git上，这个因为与版本有关，具体就不太清楚了。
最后输入

$ hexo d -g

第一次可能会弹出输入git账号密码，之后就不需要了，还有就是这步骤的命令因为要下载和上传git要久一点，耐心等待。

最后就可以通过你新建仓库的名称来访问你的博客了。
